# Dare Requirements

Source: `packages/playbook/src/index.ts`.

This file normalizes the prose-only requirements for every playbook dare into a practical planning/reference sheet. Requirements are descriptive metadata, not safety guidance. Any dare involving public, semi-public, workplace, third-party, travel, transport, or shared-property settings should be treated as requiring a legal setting, adult consent, privacy controls, and permission where applicable.

## Requirement Vocabulary

| Requirement | Meaning |
| --- | --- |
| Adult consent | Everyone involved or intentionally shown must be an adult and consenting. |
| Camera/photo | Requires taking still photos. |
| Video/audio | Requires recording video or spoken audio. |
| Private setting | Can be completed in a private controlled space. |
| Public/semi-public setting | Mentions a public, commercial, shared, or visible-to-others setting. |
| Specific location | Requires a named place such as a mall, airplane, car wash, bathroom, etc. |
| Weather/season | Requires thunder, rain, wind, snow, hot weather, or water/cold conditions. |
| Clothing/outfit | Requires a specific garment, undergarment state, costume, or outfit setup. |
| Props/items | Requires objects such as candles, food, ribbon, handcuffs, dice, phones, etc. |
| Adult toy | Requires adult toys or sexual accessories. |
| Partner/helper | Requires or optionally uses another consenting adult. |
| Stranger/service worker | Mentions interaction with a stranger, worker, driver, or delivery person. |
| Online/platform | Requires a web platform, subreddit, chat service, or messaging app. |
| Vehicle/transport | Requires car, rideshare, bus/public transport, airplane, boat, or similar. |
| Duration/timing | Requires a full day/week, repeated action, countdown, or timed sequence. |
| Safety plan | Source text explicitly implies safety, release, fire, cold, vehicle, bondage, or water risk. |
